Execute a TypeScript script that orchestrates the tools active for this request. Use this for batch operations such as loops, concurrency, filtering, and aggregation. Intermediate results stay out of the conversation context; only the returned value comes back.

The code argument is the body of an async function written in TypeScript or JavaScript:
- Call only exact tool IDs declared in the generated tools interface below.
- Every declared tool returns a ToolResult containing title, output, and metadata.
- Use Promise.all or Promise.allSettled for concurrency. At most 8 calls run at once.
- Console output is captured with a size cap and should be used only for debugging.
- Return a small aggregated object, array, or string.

Limits per execution: 50 tool calls, 8 concurrent calls, 60 seconds of active compute time, and a 256 KB result. Time waiting for tool calls is not charged against active compute time.

The nested dispatcher uses the same effective tool definitions as the parent request. A tool absent from the declarations below is unavailable. Per-tool permission checks and approvals still apply.

State does not persist between executions. Return small data directly. For larger machine-to-machine data, use the files raw-I/O API in the OS temporary directory:
```
await files.writeText("/tmp/scan-results.json", JSON.stringify(results))
const results = JSON.parse(await files.readText("/tmp/scan-results.json"))
```

Tool outputs are agent-facing text and may be formatted or truncated. Use files.readText when exact bytes are required. files.readText is restricted to the project worktree and OS temporary directory. files.writeText is restricted to the OS temporary directory; project writes must use an active declared project-writing tool so normal permissions apply.

Use tool_script when the same declared tool must be called many times, or when results need deterministic filtering, aggregation, sorting, or counting before entering the conversation.

Do not use tool_script when one direct call is sufficient, each step needs semantic judgment, results need visual inspection, or high-risk side effects should be reviewed one at a time.
